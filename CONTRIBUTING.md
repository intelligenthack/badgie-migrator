# Contributing to Badgie.Migrator

Thank you for your interest in contributing to Badgie.Migrator!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/badgie-migrator.git`
3. Create a branch: `git checkout -b my-feature`
4. Make your changes
5. Run tests: `dotnet test`
6. Commit your changes: `git commit -m "Add my feature"`
7. Push to your fork: `git push origin my-feature`
8. Open a Pull Request

## Development Setup

### Prerequisites

- .NET 8.0, 9.0, or 10.0 SDK
- A code editor (VS Code, Visual Studio, Rider, etc.)

### Building

```bash
dotnet build
```

### Running Tests

```bash
dotnet test
```

## Code Style

- Follow existing code conventions in the project
- Use meaningful variable and method names
- Keep methods focused and small
- Add tests for new functionality

## Pull Request Guidelines

- Keep PRs focused on a single change
- Update documentation if needed
- Ensure all tests pass
- Add tests for new features or bug fixes

## Reporting Issues

- Use the issue templates when available
- Provide clear reproduction steps for bugs
- Include .NET version and OS information
